Python 總複習

"""
    Python核心總複習
        1. Python 執行過程
            原始程式碼  -“編譯”->  位元組碼(.pyc)  -解釋->  機器碼
            |      第一次        |
            main.py  -- 入口代碼應當簡潔
        2. 自動化記憶體管理機制
            引用計數： 每個物件記錄被變數綁定(引用)的數量,當為0時被銷毀。
                缺點：對迴圈引用無效
            標記清除：當記憶體不足時,對整塊記憶體空間進行掃描,清空垃圾資料(整理有用的資料).
                缺點：全盤掃描記憶體,計算量大。
            分代回收：將記憶體劃分為多個區域(代),當記憶體不足時，進行升代操作.
                    將有用的資料,轉移到下一代,垃圾資料清空.
                    由於分代,所以避免了全盤掃描.
            記憶體優化：盡少產生垃圾、物件集區(減少重複物件的存在,減少物件創建與銷毀的過程)
                    可以設置分代回收參數
        3. 容器
            字串str:存儲字元編碼,不可變序列
            列表list:存儲變數,可變序列
            元組tuple:存儲變數,不可變序列
            字典dict:存儲鍵值對,可變散列
            集合set:存儲鍵,可變散列
            序列與散列：
                序列:連續的記憶體空間，有順序(索引.切片)
                散列:不連續的記憶體空間，順序(通過key)
            可變與不可變：
                可變：預留空間
                不可變：按需分配
    4. 可變與不可變對象的傳參：
            不可變資料,在函數內的修改,不影響函數外
            可變資料,在函數內的修改,可以影響函數外
        結論：傳入可變物件,修改可變物件,可以不通過返回值傳遞結果
    5. 函數參數
        實參：如何給形參傳遞資料
            位置實參：根據順序對應
                序列實參： 函數名(*序列)       【拆】
            關鍵字實參：根據名稱對應 函數名(形參名 = 資料)
                字典實參： 函數名(**字典)      【拆】
        形參：規定傳參方式
            默認參數：實參可以不傳遞
                def 函數名(形參名 = 預設值)
            位置形參：實參必須傳遞
                星號元組形參：實參數量無限  def 函數名(*args)   【合】
            命名關鍵字形參：實參必須是關鍵字實參
                def 函數名(*args,a,b)
                def 函數名(*,a,b)
                雙星號字典形參：關鍵字實參數量無限  def 函數名(*kwargs)【合】
"""
a = "悟空"  # 引用計數增加1
b = a      # 引用計數增加1
c = b      # 引用計數增加1
del a      # 引用計數減少1
b = "八戒" # 引用計數減少1
c = None  # 引用計數減少1
# 悟空因為引用計數為零,所以被銷毀.

list01 = []
list02 = []
list01.append(list02)
list02.append(list01)

# 對不可變對象的頻繁修改，要改為對可變對象的修改。
list_result = []
for i in range(100):
    list_result.append(str(i))

data01 = 100.7890
data02 = 100.7890
print(id(data01),id(data02))

def func01(p1,p2):
    # 不可變資料,在函數內的修改,不影響函數外
    p1 = 2
    # 可變資料,在函數內的修改,可以影響函數外
    p2[0] = 2

a = 1
list01 = [1]
func01(a,list01)
print(a)
print(list01)

    物件導向總複習
    1. 面向過程與物件導向：
        考慮問題從步驟(實現細節)角度出發
        考慮問題從物件(誰)角度出發
    2. 類和對象
        實例成員:變數、方法.  每個物件不同的資料與行為.
        類成員:變數、方法.  大家(共用)的資料與行為.
        靜態方法：函數.   工具
    3. 三大特徵：
        封裝：分而治之（變化點單獨定義）
        繼承：抽象 -->統一 --> 隔離(變化點)
        多態：執行變化,定義變化（重寫）
        核心理念：先用,再做.
    6. 六大原則：
        開閉原則：允許增加新功能,不能修改用戶端代碼.
        單一職責：每個類有且只有一個改變的原因
        依賴倒置：調用父,不調用子.
        組合複用：僅僅複用代碼，優先考慮組合關係,不使用繼承關係。
        裡氏替換：父出現的地方，可以使用子替換.
                 建議使用擴展重寫
        迪米特：低耦合
"""

    Python高級總複習
    1. 導入三種方式：
        import 路徑.模組 as 別名
        from 路徑.模組 import 成員

    3. 異常處理
        語法：
            try:
                可能觸發異常的語句
            except 錯誤類型1 [as 變數1]：
                處理語句1
            except 錯誤類型2 [as 變數2]：
                處理語句2
            except Exception  [as 變數3]：
                不是以上錯誤類型的處理語句
            else:
                未發生異常的語句
            finally:
                無論是否發生異常的語句
        現象：當異常發生時，程式不會再向下執行，而轉到函數的調用語句。
        目的：將異常流程(向上)轉換為正常流程(向下)
        目標：邏輯錯誤,非語法錯誤.
        人為拋出異常：快速傳遞資訊
